MATERIALS MANAGEMENT (MM) — PetabyteAi SAP Knowledge Base
==========================================================
Version: S/4HANA 2023

## OVERVIEW

MM (Materials Management) covers procurement, inventory, material master,
vendor management, and invoice verification. In S/4HANA, the core table
MKPF+MSEG is merged into MATDOC (one-table paradigm) — always query MATDOC
for goods movements in S/4, NOT MKPF/MSEG (which are now compatibility views).

## KEY BAPIs

### Material Master
- **BAPI_MATERIAL_SAVEDATA** — create/change material
- **BAPI_MATERIAL_GET_DETAIL** — read with views
- **BAPI_MATERIAL_GETLIST** — search (use sparingly; prefer CDS in S/4)
- **BAPI_MATERIAL_MAINTAINDATA_RT** — extended (tax, retail)
- **BAPI_MATERIAL_EXISTENCECHECK** — check if material exists

### Purchase Orders
- **BAPI_PO_CREATE1** — create PO (preferred, replaces BAPI_PO_CREATE)
- **BAPI_PO_CHANGE** — change existing PO
- **BAPI_PO_GETDETAIL1** — read PO with schedule lines + history
- **BAPI_PO_RELEASE** — trigger release strategy
- **BAPI_PO_RESET_RELEASE** — revoke release

### Purchase Requisitions
- **BAPI_PR_CREATE** — create PR
- **BAPI_REQUISITION_CREATE** (older) — avoid, use BAPI_PR_CREATE
- **BAPI_PR_CHANGE** ||
- **BAPI_PR_GETDETAIL** ||
- **BAPI_PR_RELEASE** ||

### Goods Movement
- **BAPI_GOODSMVT_CREATE** — create material document (GR, GI, Transfer)
- **BAPI_GOODSMVT_CANCEL** — cancel material document
- **BAPI_GOODSMVT_GETITEMS** — read items (use MATDOC in S/4)

### Invoice Verification
- **BAPI_INCOMINGINVOICE_CREATE** — create MIRO invoice
- **BAPI_INCOMINGINVOICE_PARK** — park for later
- **BAPI_INCOMINGINVOICE_RELEASE** — release blocked

### Vendor (pre-S/4)
- **BAPI_VENDOR_CREATE** — create vendor (LFA1/LFB1/LFM1)
- **BAPI_VENDOR_GETDETAIL** ||
- In S/4: use **CVI_VENDOR_CREATE_FROM_BP** or BP transaction

## CODE EXAMPLE — CREATE PO

```abap
DATA: ls_header   TYPE bapimepoheader,
      ls_headerx  TYPE bapimepoheaderx,
      lt_items    TYPE TABLE OF bapimepoitem,
      lt_itemsx   TYPE TABLE OF bapimepoitemx,
      lt_sched    TYPE TABLE OF bapimeposchedule,
      lt_schedx   TYPE TABLE OF bapimeposchedulx,
      lt_return   TYPE TABLE OF bapiret2,
      lv_po_num   TYPE bapimepoheader-po_number.

ls_header-comp_code  = '1000'.
ls_header-doc_type   = 'NB'.         " Standard PO
ls_header-vendor     = '0000100001'. " 10-digit
ls_header-purch_org  = '1000'.
ls_header-pur_group  = '001'.
ls_header-currency   = 'EUR'.

ls_headerx-comp_code = 'X'.
ls_headerx-doc_type  = 'X'.
ls_headerx-vendor    = 'X'.
ls_headerx-purch_org = 'X'.
ls_headerx-pur_group = 'X'.
ls_headerx-currency  = 'X'.

APPEND VALUE #( po_item = '00010' material = 'MAT001'
                plant = '1000' quantity = 10 net_price = '100.00' po_unit = 'PC' )
        TO lt_items.
APPEND VALUE #( po_item = '00010' material = 'X' plant = 'X'
                quantity = 'X' net_price = 'X' po_unit = 'X' )
        TO lt_itemsx.

APPEND VALUE #( po_item = '00010' sched_line = '0001'
                delivery_date = sy-datum quantity = 10 ) TO lt_sched.
APPEND VALUE #( po_item = '00010' sched_line = '0001'
                delivery_date = 'X' quantity = 'X' ) TO lt_schedx.

CALL FUNCTION 'BAPI_PO_CREATE1'
  EXPORTING  poheader       = ls_header
             poheaderx      = ls_headerx
  IMPORTING  exppurchaseorder = lv_po_num
  TABLES     return         = lt_return
             poitem         = lt_items
             poitemx        = lt_itemsx
             poschedule     = lt_sched
             poschedulex    = lt_schedx.

IF line_exists( lt_return[ type = 'E' ] ) OR
   line_exists( lt_return[ type = 'A' ] ).
  CALL FUNCTION 'BAPI_TRANSACTION_ROLLBACK'.
ELSE.
  CALL FUNCTION 'BAPI_TRANSACTION_COMMIT' EXPORTING wait = 'X'.
ENDIF.
```

## CODE EXAMPLE — GOODS RECEIPT FOR PO (101)

```abap
DATA: ls_header TYPE bapi2017_gm_head_01,
      ls_code   TYPE bapi2017_gm_code,
      lt_items  TYPE TABLE OF bapi2017_gm_item_create,
      lt_ret    TYPE TABLE OF bapiret2,
      lv_mat    TYPE bapi2017_gm_head_ret-mat_doc.

ls_header-pstng_date = sy-datum.
ls_header-doc_date   = sy-datum.
ls_code-gm_code      = '01'. " MB01 - GR for PO

APPEND VALUE #(
  move_type     = '101'
  po_number     = '4500000001'
  po_item       = '00010'
  material      = 'MAT001'
  plant         = '1000'
  stge_loc      = '0001'
  entry_qnt     = 10
  entry_uom     = 'PC'
  mvt_ind       = 'B'
) TO lt_items.

CALL FUNCTION 'BAPI_GOODSMVT_CREATE'
  EXPORTING  goodsmvt_header = ls_header
             goodsmvt_code   = ls_code
  IMPORTING  materialdocument = lv_mat
  TABLES     goodsmvt_item   = lt_items
             return          = lt_ret.
```

## RELEASE STRATEGIES (PR / PO)

- Configured in SPRO: **MM → Purchasing → Purchase Order/Requisition → Release Procedure**
- Tables: **T16FS** (release strategy), **T16FG** (release groups), **T16FC** (release codes)
- Programmatic release: `BAPI_PO_RELEASE` / `BAPI_PR_RELEASE`
- Check release status via EKKO-FRGKE ('R' = released, ' ' = blocked)

## COMMON INTEGRATION POINTS

- **MM ↔ FI**: Invoice posting creates FI doc (BKPF/BSEG). Account determination via OBYC.
- **MM ↔ CO**: GI to cost center posts actual costs (COEP).
- **MM ↔ SD**: Stock transfer order (STO) ties intercompany flow.
- **MM ↔ WM/EWM**: Goods movement triggers warehouse task (in EWM: /SCWM/ tables).

## S/4HANA CHANGES TO WATCH

1. **Material number extended** to 40 chars (was 18). Use MATNR_EXTERNAL / MATNR_INTERNAL helpers.
2. **BP is mandatory** — vendor creation via XK01 is deprecated, use BP tcode or CVI_*.
3. **MATDOC** replaces MKPF+MSEG. Direct SQL on MKPF/MSEG still works (CDS-based views) but is slower; use MATDOC_EXTRACT CDS view.
4. **MM-IV (Classic) → New Invoice Verification**: LIV (MIRO) is the only supported flow.
5. **Output management**: Post-1909, prefer BRF+/Adobe over NAST/SMARTFORMS for new dev.
