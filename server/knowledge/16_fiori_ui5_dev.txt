========================================================
Version: UI5 1.120 LTS / Fiori Elements / BAS 2024

## OVERVIEW

Fiori = UX guideline + apps. UI5 = framework. Three dev modes:
1. **Fiori Elements** — metadata-driven, no/low-code. Use when OData service has full annotations.
2. **Freestyle UI5** — full control (XML views + JS controllers).
3. **CAP + UI5** — Cloud Application Programming Model full-stack app.

Recommended tooling: **SAP Business Application Studio (BAS)** on BTP, or
**VS Code + SAP Fiori Tools** extensions.

## CONTROLLER PATTERN

```javascript
sap.ui.define([
    "sap/ui/core/mvc/Controller",
    "sap/ui/model/Filter",
    "sap/ui/model/FilterOperator",
    "sap/m/MessageToast"
], function (Controller, Filter, FilterOperator, MessageToast) {
    "use strict";

    return Controller.extend("com.petabyte.order.controller.Main", {

        onInit: function () {
            const oRouter = this.getOwnerComponent().getRouter();
            oRouter.getRoute("main").attachPatternMatched(this._onMainMatched, this);
        },

        _onMainMatched: function () {
            // refresh list-level model on route entry
            this.byId("orderTable").getBinding("items").refresh();
        },

        onSearch: function (oEvent) {
            const sQuery = oEvent.getParameter("query") || "";
            const oBinding = this.byId("orderTable").getBinding("items");
            if (!sQuery) { oBinding.filter([]); return; }
            const aFilters = [
                new Filter("CustomerName", FilterOperator.Contains, sQuery),
                new Filter("OrderID",      FilterOperator.EQ, sQuery)
            ];
            oBinding.filter(new Filter({ filters: aFilters, and: false }));
        },

        onItemPress: function (oEvent) {
            const oCtx = oEvent.getSource().getBindingContext();
            const sId  = oCtx.getProperty("OrderID");
            this.getOwnerComponent().getRouter().navTo("detail", { orderId: sId });
        },

        onSave: async function () {
            const oModel = this.getView().getModel();   // ODataModel V2
            try {
                await new Promise((resolve, reject) => {
                    oModel.submitChanges({
                        success: resolve,
                        error: reject
                    });
                });
                MessageToast.show(this.getResourceBundle().getText("saveSuccess"));
            } catch (e) {
                MessageToast.show("Save failed: " + (e.message || "unknown"));
            }
        },

        getResourceBundle: function () {
            return this.getOwnerComponent().getModel("i18n").getResourceBundle();
        }
    });
});
```

## ODATA V2 vs V4 CHEAT SHEET

| Feature | V2 | V4 |
|---------|----|----|
| Model class | sap.ui.model.odata.v2.ODataModel | sap.ui.model.odata.v4.ODataModel |
| Read with filter | model.read("/OrderSet", { filters: [] }) | oListBinding.filter([]) + requestContexts() |
| Create | model.create("/OrderSet", data, callbacks) | oListBinding.create(data) |
| Delete | model.remove(path, callbacks) | oContext.delete() |
| Batch default | off (set setUseBatch(true)) | on (always batched) |
| Two-way binding | via oData.setDefaultBindingMode("TwoWay") | Property-level bindings: { path: "X", $$ownRequest: true } |
| Function Import | model.callFunction("/ACTION", …) | oContext.execute() (bound actions) |
| Draft support | no native | yes via `com.sap.vocabularies.Common.v1.DraftRoot` |

**No controller** written for list report; customize via "extension points"
and `controllerExtensions` in manifest.


## DEPLOYMENT

### On-premise S/4HANA
1. `ui5 build` → dist/
2. `npx @ui5/cli build --clean-dest`
3. Deploy via **UI5 Deployer** (`@sap/ux-ui5-tooling`):
   ```bash
   npx fiori deploy --config ui5-deploy.yaml --archive-path dist.zip
   ```
4. Publishing → /UI2/FLPCM_CUST assigns to catalog/group → PFCG assigns catalog to role.

### BTP (Launchpad Service / HTML5 Repo)
1. `npm run build:cf` or `npm run build:mta`
2. `mbt build` → `*.mtar`
3. `cf deploy target.mtar`
4. Configure **Site Manager** to expose as tile.

## PERFORMANCE TIPS

- Enable **UI5 Cache Buster** (`"sap-ui-cachebuster-info": "/sap-ui-cachebuster/"`)
- Bundle with `ui5 build --all` → preload.js cuts round-trips
- Use `$select=` in OData requests to fetch only needed fields
- For large lists: enable table `growing` + paging; avoid `items="{path: '/Huge', length: 9999}"`
- V4: set `$$operationMode: "Server"` so sort/filter go to backend
- Avoid XML view with >300 controls — lazy-render via `sap.ui.xmlfragment`

## I18N RULES

- Keys in upper_snake: `BTN_SAVE`, `MSG_SUCCESS`, `COL_AMOUNT`
- Fallback is `i18n.properties` (English). Thai: `i18n_th.properties`.
- Escape apostrophes with two: `It''s` not `It's`.
- Parameters: `{0}, {1}` — pass via `oBundle.getText("KEY", [v0, v1])`.

## ACCESSIBILITY (WCAG via Fiori)

- Always set `label` on inputs (or aria via `labelFor`)
- High-contrast theme `sap_horizon_hcb` covers visual needs
- Keyboard: never trap focus — use `sap.m.Dialog` with `initialFocus`
- Live regions: `sap.m.MessageStrip` with `type="Information"` announces

## TROUBLESHOOTING

| Symptom | Likely Cause |
|---------|--------------|
| "Cannot read ... of undefined" in controller | Model not yet loaded — guard with `attachDataReceived` or use async/await |
| 403 CSRF failed | Missing `X-CSRF-Token`. V2 model auto-fetches via HEAD `?sap-client=X`. Check destination in BAS. |
| Blank screen after deploy | Component.js path wrong in manifest.json / incorrect Component-preload build |
| OData 404 in UI | Destination URL or `sap-client` mismatch; test in browser with creds |
| Slow initial load | Component-preload not built; no cache buster |
