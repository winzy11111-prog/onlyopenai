## OVERVIEW

**SPRO** = SAP Project Reference Object = the Implementation Guide (IMG).
Functional consultants configure SAP modules through SPRO — no coding,
but changes are recorded in **Customizing requests** and transported.

Developers should understand **enough** SPRO to:
1. Find where a behavior is configured (e.g. number ranges, pricing, output)
2. Add enhancements at the right layer (User Exit vs BAdI vs Enhancement Spot)
3. Know which **customizing tables** (T-tables) back which settings

## NAVIGATING SPRO

1. Tx `SPRO` → "SAP Reference IMG" button
2. Expand tree: **SAP Customizing Implementation Guide → <Module> → …**
3. Click the **activity** (clock icon) to jump to the config screen
4. Use **Activity Info** (📘) to read SAP docs for that node
5. Use search: `Utilities → Find` with keyword

## ENHANCEMENT TECHNIQUES (for functional → technical handoff)

### Pyramid (newest first)
1. **BAdI (Business Add-In)** — OO-based, multi-implementation.
   Tx: **SE18** (defn) / **SE19** (impl).
2. **Enhancement Spots / Implicit Enhancements** — explicit source-code hooks.
3. **Customer Exit (CMOD / SMOD)** — legacy, still works. Function modules `EXIT_*`.
4. **User Exit** — form routines (SD: USEREXIT_MOVE_FIELD_TO_VBAK in MV45AFZZ).
5. **Modification** — direct source edit (requires SSCR key, logged in CMOD).

Rule: **BAdI > Enhancement > Customer Exit > User Exit > Modification**
Prefer higher tiers; lower tiers are upgrade-risky.

### Finding the Right BAdI
```abap
" Put breakpoint in CL_EXITHANDLER=>GET_INSTANCE
" Run the transaction — every BAdI lookup stops here
" Check parameter EXIT_NAME to see what's called
```

Or use tx `SE84` → "Business Add-Ins" → search by text.

## OUTPUT MANAGEMENT

### Classic NAST / Condition-Based Output
- Tx `NACE` — application + output type + condition records
- Print program + SmartForm/SAPscript link
- Triggered via **NAST** table entries + RSNAST00

### Modern (S/4 1709+): Output Management
- **Adobe Forms** or **Margin print**
- Configured via **BRF+** decision tables
- SPRO: *Cross-Application Components → Output Control*
- Adoption: SAP strongly recommends for new dev

## TRANSPORT OF CUSTOMIZING

- Customizing requests are **client-specific** (unlike Workbench requests).
- Check target client in **SCC4** — must allow customizing changes.
- View request contents: SE10 → double-click request → objects.
- Some entries only transport via **manual SM30 transport** button or via
  **BC Set** (tx `SCPR3`).

## COMMON FUNCTIONAL ERROR PATTERNS

| Symptom | Usually caused by |
|---------|-------------------|
| "No posting period variant" | OBBO missing / period closed |
| "Account XXX not allowed for doc type" | OBA7 field status vs FS00 mismatch |
| "No account assignment possible" | OBYC missing key for valuation class |
| "Pricing error mandatory condition XR1 missing" | V/08 or master record VK11 not maintained |
| "No authorization for company code" | PFCG role missing F_BKPF_BUK |
| "Output type determined, no printer assigned" | NACE condition record missing output device |
| "Update terminated" (SM13) | Config inconsistency or commit after custom BAPI — check update dump in SM13 |

## WHEN DEV TALKS TO FUNCTIONAL CONSULTANT

Ask **four questions** before changing anything:
1. **Which SPRO node?** (ask them to open and show path)
2. **Which transport?** (cust vs workbench)
3. **Which tables?** (so you can read defaults in code)
4. **Which enhancement point exists?** (BAdI > User Exit) — avoid modification

For any "we need to add a field", confirm:
- Append structure on standard table (dictionary-level) **or** CI_* include
- Screen enhancement via screen exit / Web Dynpro extension / RAP extension
- Data flow: is it persisted (DB), calculated (CDS), or display-only (UI)?
