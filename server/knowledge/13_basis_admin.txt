## OVERVIEW

BASIS = SAP system administration. Covers user/role management, transports,
background jobs, performance monitoring, system logs, and kernel operations.
For developers: focus on **authorization objects**, **debugging dumps**, and
**background job scheduling**. For admins: everything else.

### AUTHORITY-CHECK Pattern

```abap
AUTHORITY-CHECK OBJECT 'S_TABU_DIS'
  ID 'DICBERCLS' FIELD '&NC&'    " authgroup
  ID 'ACTVT'     FIELD '03'.      " 03=display, 02=change

IF sy-subrc <> 0.
  MESSAGE e001(00) WITH 'No authorization'.
ENDIF.
```

Always check sy-subrc:
- 0  = authorized
- 4  = missing auth
- 12 = object not in user buffer
- 24 = field value missing in check

### Transport Flow
**DEV → QAS → PRD** (typical 3-tier)
1. Change recorded in Workbench Request (cross-client) or Customizing Request (client-specific).
2. Release task → request → exported from DEV (auto: files to /usr/sap/trans/).
3. Import to QAS via STMS_IMPORT. Test.
4. Import to PRD via STMS_IMPORT. Monitor RC.

### Return Codes
| RC | Meaning |
|----|---------|
| 0  | Success |
| 4  | Warnings (review) |
| 8  | Errors (generation failed) |
| 12 | Severe errors (objects missing) |
| 16+ | Fatal — abort and investigate |

### Job States
- **Scheduled** — defined but not released
- **Released** — awaiting start condition
- **Ready**    — ready to run, waiting for work process
- **Active**   — running
- **Finished** — completed OK
- **Canceled** — aborted (check ST22 / job log)

### Schedule a Job in ABAP

```abap
DATA: lv_jobcount TYPE tbtcjob-jobcount,
      lv_jobname  TYPE tbtcjob-jobname.

lv_jobname = 'ZJOB_NIGHTLY_CLEANUP'.

CALL FUNCTION 'JOB_OPEN'
  EXPORTING jobname  = lv_jobname
  IMPORTING jobcount = lv_jobcount.

SUBMIT zreport_cleanup
  WITH p_date = sy-datum
  VIA JOB lv_jobname NUMBER lv_jobcount
  AND RETURN.

CALL FUNCTION 'JOB_CLOSE'
  EXPORTING jobcount  = lv_jobcount
            jobname   = lv_jobname
            sdlstrtdt = sy-datum
            sdlstrttm = '230000'.
```

### Lock Object Usage in ABAP

```abap
CALL FUNCTION 'ENQUEUE_E_MATERIAL'   " generated from lock object
  EXPORTING matnr = 'MAT001'
  EXCEPTIONS foreign_lock = 1 system_failure = 2 OTHERS = 3.

IF sy-subrc = 1.
  MESSAGE |Locked by { sy-msgv1 }| TYPE 'E'.
ENDIF.

" ... change logic ...

CALL FUNCTION 'DEQUEUE_E_MATERIAL'
  EXPORTING matnr = 'MAT001'.
```

## PERFORMANCE DIAGNOSIS WORKFLOW

1. **ST03N** → identify top response-time transactions.
2. **ST05** → trace that tx: toggle SQL trace on, reproduce, toggle off, analyze.
3. **SAT** → ABAP-level profiling (line-by-line CPU time).
4. **ST22** → check for concurrent dumps.
5. **ST04** → DB-side expensive SQL.
6. **SM66** → if long-running work processes, check semaphores/locks.

```abap

  EXCEPTIONS system_failure = 1 communication_failure = 2 OTHERS = 3.

  MESSAGE |RFC dest ECC_PROD unreachable: { sy-msgv1 }| TYPE 'E'.
ENDIF.
```

## COMMON BASIS HEALTH CHECKLIST (daily)

- **SM21** — no red entries
- **ST22** — review dumps, link to defects
- **SM37** — canceled jobs
- **SM13** — no stuck updates
- **ST02** — no buffer swaps > 10k/day
- **SM12** — no locks older than session timeout
- **ST06** — CPU/memory/disk within threshold
- **RZ20** — no red CCMS alerts
- **STMS_IMPORT** — clear queue for QAS/PRD
